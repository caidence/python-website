Booleans

Probably the most simple to understand data type is a Boolean. A Boolean can only have two possible values: true or false. That's it, really! They may be simple, but they are very powerful. Other types in Python also have a Boolean value of true or false. I intentionally waited to introduce the Boolean data type until this section because they are commonly used with comparison statements.

>>> bool("")
False
>>> bool("Hi!")
True
>>> bool(1.9)
True
>>> bool(0)
False
>>> bool([])
False
>>> bool([1, 7])
True

Like I mentioned earlier, objects in Python evaluate to either true or false. To test whether an object evaluates to true or false, we can place the object inside the brackets in the bool( ) function. Python will then return the Boolean value of that object. Generally, empty objects evaluate to false and objects that contain data evaluate to true. You will see this in the following examples.

On the first line of the example, we test to see the Boolean value of an empty string. Because the string contains nothing, the test evaluates to false. Any empty data type will evaluate to false unless you program it otherwise. Next, we test the Boolean value of a string with characters in it, because the string is not empty, it evaluates to true. As you can see with the next two tests, any number that is not zero will evaluate to true. The number can be a float, positive or negative, as long as the number is not zero, its Boolean value will be true. After that, we test the Boolean value of an empty list, which of course evaluates to false because it is empty. Finally, we test the Boolean value of a list with items in it. Once again, this will evaluate to true because the list is not empty. Pretty simple!

Comparisons

1. ==  # Equality
2. !=  # Not equal to
3. <  # Less than
4. >  # Greater than
5. <=  # less than or equal to
6. >=  # Greater than or equal to

Comparisons are statements that either evaluate to true or false. They are basically a test to determine what to do next in the program. In this section, I will introduce some of the most common comparison statements. Here are some examples of a few of the most commonly used.

These statements are used to compare two objects. The objects can be any data type, but are most commonly the same data type. In the following sections, I will go into more detail with examples to demonstrate how comparison tests work.

<, >, <=, >=

>>> 1 < 2
True
>>> 77 > 100
False
>>> .33 <= .33
True
>>> 1.9 >= 2
False

Comparisons evaluate whether an expression is true or false. With comparisons, we essentially are testing two or more values against each other to get the Boolean value of the expression. In the first example, we test to see if one is less than two. This is true, making the return value of the test true. Next, we test to see if seventy-seven is greater than one-hundred. This, of course, is false, making the return value of the test false. The rest of these examples are pretty self-explanatory. In the following examples, I will demonstrate how to use these tests with strings, lists, and tuples.

>>> [1, 2] > [2]
False
>>> [1, 2, 0] > [1, 2]
True
>>> (1, 2) > (2)
False
>>> (1, 2, 0) > (1, 2)
True

When you use these statements on lists and tuples, things get a little more interesting. The operator does not compare the length of the lists or tuples, as you might expect. Nor does it compare the sum of the elements in the object. When you use these operations on lists and tuples, it will compare the item at the first index in the objects. This is why in the first example the output is false. Python compares the first item in each list and since two is greater than one, the list on the right is greater than the list on the left. What if both of the first numbers are the same? Then Python will compare the second numbers. If the second number of each list is the same, then Python will compare the third number and so on. You can see how this works in the second example where two lists have the same first and second number, 1 and 2. Because the first list has a third number and the second does not, the first list is considered to be greater than the second. The next two examples demonstrate how this behavior is exactly the same with tuples.

>>> "red" < "rock"
True
>>> "a" > "z"
False

Like with lists and tuples, when you use these operations on two strings the values compared are the first characters in the strings. If the first characters are the same in both strings, then the second character is compared and so on. How can we compare strings if their values are not numerical? The answer is Unicode. I am not going to explain in depth how Unicode works here, but basically every character has a number associated with it so that the computer can identify what the character is. We can view what number value a character has by using the ord function. Just know, that when you use these operations on strings, the character's Unicode value is what is being compared.

>>> ord("a")
97
>>> ord("!")
33
>>> ord("A")
65

The ord function can only accept one character at a time, otherwise you'll get an error. The function will return the number value of the Unicode mapping of the character passed to it. The numerical value mapped to the character "a" is 97 in this particular coding scheme. The examples demonstrate this and a few more Unicode mappings.

==, !=

These are probably the two most common comparison checks you will use in Python. The double equal sign (==) checks for equality. What this means is: checking to see if two pieces of data hold the same value. == will return true as long as the values stored in the data are equal to each other otherwise, the expression will evaluate to false. The not equal operator (!=) is just the opposite of ==. != checks to see if two items are not equal, and will return true so long as the objects do not hold the same values. In Python, the exclamation point (!) means not, which is why we call this operator (!=) not equal. Also, if two objects hold the same data, but are not the same data type, then they are not equal. For example, say we have a one item list and a one item tuple. We use the expression [1] == (1,). The return value will be false even though they both contain a 1. This is because the data types are different.

>>> # Equality tests ==
>>> [1] == (1,)
False
>>> "Hello" == 'Hello'
True
>>> 1 == 1.1
False
>>> # Not equal to tests !=
>>> 1 != 1
False
>>> 1 != 2
True
>>> True != False
True

Just a quick rundown of what is going on in this example. In the first few lines, we test for equality using ==. Once again, this evaluates whether the values are the same or not. If the values are the same, the expression will return true, the expression returns false otherwise. In the last half of the example, we are testing to see if the values are not by equal using !=. If the values are not equal, the expression returns true, the expression returns false otherwise. != is just the inverse of ==.
