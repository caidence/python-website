Conditional Statements

if

Simply put, if statements are a compound statements that dictate the logic of a program. if is used to see if a statement evaluates to True, if the statement, in fact, does evaluate to True, the indented code inside of the if block will run. If the statement evaluates to False, Python will skip the code within the code block of the if statement and continue executing.

1. if True:
2.     print(True)
3.
4. print('This prints no matter what')

There is a lot that you should be aware of in this statement. The if statement should end with a colon (:) and the code inside the code block should be indented by four spaces. It is not required that you indent your code blocks by four spaces, but it must be indented. It is just a strong Python convention to indent blocks of code by four spaces. I suggest you follow this convention.

Usually, you will be evaluating a statement that will be either True or False, but I wanted to show you exactly how the if statement works by forcing the condition to evaluate to True. The output of this statement will always be True. Because the if statement evaluates to True, the indented code inside of the if block will run, which will print True. If the statement was False, then the code inside of the block would not run and the program would continue to execute. The print statement on line four will execute, regardless of what the if statement evaluates to because it is not indented inside the if block.

1. age = int(input("Age: "))
2. if age > 100:
3.     print("You're pretty old!")

This is a more practical example of using an if statement. This program first will ask the user for their age and store the value in a variable called age. The program then evaluates whether the age variable is greater than 100 or not by using the expression if age > 100. If and only if (iff) this statement evaluates to true, then the code inside the if block will run. In this case, if the statement is True, that means the user is over 100 years old and the program will tell the user that they are old.

elif

Adding onto the if statement, we have the elif statement. We use elif if we are evaluating multiple conditions. elif works exactly the same as if, it evaluates a statement, if the statement evaluates to true, the code indented inside of the elif block will run. Elif can only come after an if statement and the elif statement will only be evaluated if the if statement evaluates to false. You can use as many elif statements as you want, just remember that order does matter when it comes to elif statements. After one elif statement evaluates to true, none of the elif statements after the true statement will even be evaluated.

1. age = int(input("Age: "))
2. if age > 100:
3.     print("You're pretty old!")
4. elif age < 18:
5.     print("Dang you're young")
6. elif age == 21:
7.     print("You're old enough to drink!")

Building on our age example from earlier, this example will ask a user to input their age and will still tell the person that they are old if their age is over 100. What is different about this program is that there is the added logic, testing for multiple conditions by using multiple elif statements if the first if statement evaluates to false. If the age input to the program is less than 100, the program then checks to see if the first elif statement evaluates to true, if it does, the code indented inside of it will run and the following elif statements will not be evaluated. However, if it evaluates to false, the next elif condition is checked and so on.

else

Else can only appear after all if and elif statements. This is essentially a "failover statement," if you will. This statement does not need an expression to evaluate because it only runs if all if and elif statements evaluate to false. The else statement needs its code indented just like the if and elif statements.

1. name = input("Whats your name? ")
2. if name == "Aaron":
3.     print("Hi Aaron!")
4. else:
5.     print("You're not Aaron!")

This program will ask a user for their name. If the user's name is "Aaron", the program will greet them by saying "Hello Aaron!". If the user's name is anything besides "Aaron", the program will say "You're not Aaron!". The else statement does not need to evaluate an expression because if the if statement evaluates to false, the else block will run no matter what.

and, or

Very common logical operators used with if clauses are the and and or keywords. These keywords allow us more granular control over the logic of our programs. These operators are used to evaluate more than one condition in a single statement. The and keyword will return true iff both conditions are true. If either condition evaluates to false, the and expression will evaluate to false, too. The or keyword will return true so long as one of the expressions evaluates to true. The syntax of these statements looks like (condition 1) and (condition 2), (condition 1) or (condition 2).

1. True and True  # Returns True
2. True and False  # Returns False
3. False and True  # Returns False
4. False and False  # Returns False
5.
6. True or True  # Returns True
7. True or False  # Returns True
8. False or True  # Returns True
9. False or False  # Returns False

These are not really practical examples of how to use and and or. These are more or less to demonstrate the logic behind these two keywords. You will often hear an example like this called a logic table or truth table. Next, we will look at some more real-world-like examples of how to use and and or.

You probably would not want to write a program like this, but it is a much more practical example than the previous ones. This next program will prompt the user for a username and a password. It will only grant the user access if the username and password match what they should be. Here, we want to use the and keyword to check the conditions. The username must be correct and the password must be correct.

1. username = input("Username: ")
2. password = input("Password: ")
3. if username == "Bill" and password == "secret":
4.     print("Access granted.")
5. else:
6.     print("Access denied.")

This program begins by asking a user for their username password. Next, Python checks to see if the username equals "Bill" and if the password equals "secret". If so, then access is granted to the user. If either the username or the password does not equal what it should, the program will execute the else block, which will print "Access denied." to the screen.

Building on our last example, lets add an or keyword to our program. Let's make the program print access granted if the username is either "Bill" or "Steve". We'll make the program grant access to either Bill or Steve as long as the password is still "secret". We can do this by putting an or keyword in the first conditional statement on the left side of the and keyword.

1. username = input("Username: ")
2. password = input("Password: ")
3. if username == "Bill" or username == "Steve" and password == "secret":
4.     print("Access granted.")
5. else:
6.     print("Access denied.")

In this example, the user's name can be either Bill or Steve. The password has to still be "secret" no matter which username is typed. Remember, on either side of or you must explicitly write an expression comparing the name to username. It is a common mistake to leave out the second expression and just write another string. You must do this(username == "Bill" or username == "Steve") not this (username == "Bill" or "Steve"). The second will not work and I will explain why in the next paragraph. If you use the second, the username can be anything as long as the password is "secret", that's not what we want!

1. # This won't work, common mistake
2. username = input("Username: ")
3. password = input("Password: ")
4. if username == "Bill" or "Steve" and password == "secret":
5.     print("Access granted.")
6. else:
7.     print("Access denied.")

Remember how data types have a Boolean value? Remember how strings in particular will always evaluate to true as long as they are not empty? The expression (username == "Bill" or "Steve") will evaluate to true no matter what the username is. This is because or evaluates the Boolean value of expressions or data types, not the content of the data type. Because the second expression that or evaluates ("Steve") is not an expression, or will evaluate the Boolean value of the data type. Strings with characters in them always evaluate to true. "Steve" is a string with characters, so it will evaluate to true every time. Since or only needs one of its values to evaluate to true to make the expression true, and because "Steve" will evaluate to true every time, this will make the or expression evaluate to true every time! This means that you could input "Bob" as the username and still get granted access if you input the correct password.
