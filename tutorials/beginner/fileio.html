The Basics

With Python, we can read from and write to files that reside on the local computer. We create a file object by using the open built in function, passing the file location as a first argument, and the mode to open the file in as the second argument. This gives the ability to read and or write to/from a file.

Opening Files

Like I said previously, we can use the open function with an external file name followed by a processing mode. The call to open uses the processing mode argument to determine what methods that we can call on the file object for data transfer.

>>> file = open(filename, mode)
>>> file.method()

The first argument is the name of the file to open. This can be an absolute or relative path to the file. If no path is present, the file is assumed to be in the same directory as the program. The second argument is what mode that you want to open the file in. This determines what methods you can call on the file object. By default, if no mode is passed the file object will be opened in read mode "r". The most common modes are read "r", write "w", and append "a". Write, will overwrite the contents of a file. If you want to add to the file rather than overwrite it, open the file in append mode. You can also read from, and write to binary files. To read from a binary file, list "rb" as the mode argument in the open function. Likewise, if you wish to write to a binary file, use "wb" as the mode argument.

Writing to Files

1. file = open('file.txt', 'w')
2. file.write('Line 1\n')
3. file.write('Line 2\n')
4. file.close()

First, we open "file.txt" for writing and store the file object in a variable called file. Next, we write the string "Line 1\n" to the file object. Remember that "\n" is the newline character. On the next line, we write another string to the file object, "Line 2\n". Finally, we close the file by calling the close method on the file object. Always remember to close a file once you're done with it! Note, if you open a file with "w", the contents of the file will be overwritten, open a file with "a" if you wish to add to the existing file contents.

Reading from Files

The example in this subsection assumes that the file "file.txt" has already been written to by the example in the previous subsection.

>>> file = open('file.txt', 'r')
>>> file.read()
'Line 1\nLine 2\n'
>>> print(file.read())
Line 1
Line 2

We start by creating a file object in read mode. We can read the contents of the file into a string with the read method. As you can see, the new line characters will literally be included in the string unless we actually print it. By default, the read method will read the entire contents of the file into a string, this can be a problem for very large files. We'll look at a way to work around this in a later tutorial. A neat trick is something called a file iterator. This will print the lines of a file one by one.

1. # File iterator
2. for line in open('file.txt', 'r'):
3.     print(line, end='')

File iterators are nice because they will read and print one line of the file on each loop iteration. You do not have to do anything special for this, file iterators just do it automatically! The syntax is very readable, they are easy to code, and they are very memory efficient.

File Saftey

You should always close your files when you are done using them. This can be hard to remember sometimes, but it is very important. If you don't close your files, it can cause your program to crash and/or lose data from your file. What if an error occurs before you get the chance to close a file? Unfortunately, this may cause data loss and unexpected behavior from your program. Fortunately, however, there is a safer way to open files than the way that we have been using. The syntax is with open('file') as variable:. This syntax is what we call a context manager in Python. If you open your files this way, your file will close automatically when you are done with it or if the program errors out. Here is how to code some of the previous examples with this safe syntax.

1. with open('file.txt', 'w') as file:
2.     file.write('Line 1\n')
3.     file.write('Line 2\n')
4.
5. with open('file.txt', 'r') as file:
6.     print(file.read())

As you can see, the with statement is a block of code. All statements that you want to execute with the open file object are indented within the code block. The file will close automatically after the code block runs. The "as file" part of the with header is naming the file object "file". The block is basically saying "with this open file named as this, do this block of code."
