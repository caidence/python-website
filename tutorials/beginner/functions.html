The Basics

Functions are one of the most common paradigms used with Python (appropriately, called functional programming). Functions are a way to package logic so that you can use it at multiple places in your program. We can run a function by calling it, which we will get to later. Functions are objects in Python. In fact, you can treat them just like any other object, even passing them to other functions!

def

We can define a function by using the def keyword followed by the function name then a pair of parentheses, and finally a colon. If we were to create a function called my_function the syntax would look like this def my_function():. def is short for define, which is appropriate because we call this statement defining a function. Then we indent all of the code that we want to run when we call the function under the function definition.

>>> def my_function():
...     print('This is my funciton')
>>> my_function()
This is my funciton
>>> my_function()
This is my funciton

This one of the most basic forms of a function. The function that we defined (my_function) will print "This is my function" every time that we call it. To call a function, we simply write the function name, followed by a set of parentheses. As you can see, we can reuse the code within the function simply by calling the function a second time. There is no limit on how many times that you can call a function, you can call it as many/few times as you want.

Return

Often times, your functions will return some value. This is useful for storing the output of the function in a variable. We will cover this more in the next tutorial section, but for now let's just focus on the return keyword. return will pass a return value back to where it was called from, the program then begins to execute again from where it was before it called the function. In programming, this this is referred to as "returning control from the function back to the main program."

>>> def test():
...     return 1 * 1
>>> test()
1
>>> print('This is after the function')
This is after the function

This is not a very useful program. It is mostly to demonstrate that the return statement will return a value to the part of the program that called the function and then the program will continue to execute after the function call. All this function does is return the value of the expression 1 * 1 to the part of the program that called it.
