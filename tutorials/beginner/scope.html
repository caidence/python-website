What is scope?

Whenever you use a name in a program, Python will look up that name in what is called a namespace. When we talk about scope, what we are really referring to is the name's relation to the part of the code that it resides in. Python uses a name's place of assignment to associate it with a particular namespace.

Functions are a great of example of what we call a local namespace. This means that variables created here cannot be used outside of the local function. You cannot access names declared within a function from namespaces outside of the function. Also, names declared in the local function namespace will not clash with names declared in namespace outside of the function, even if the same names are used in more than one namespace. A namespace that resides inside of another namespace can use names declared in its parent namespace but not the other way around.

In this tutorial, we are going to learn about the four namespaces built in, global, nonlocal, and local.

Global, Built in

When we talk about global namespace, we are talking about names that reside at the top-level of the enclosing module. Every module (collection of files) is its own global scope. This means that variables created here live at the top-level of the module. Any namespace declared within the module can access the global namespace. This is commonly where functions and classes live.

1. x = 10   # Global namespace
2. def func1():
3.     print(x)
4. def func2():
5.     return func1()

In the example, the variable x is in the global namespace. It is not declared inside of a function or class, meaning that it can be accessed by all functions and classes in the same module. func1 and func2 also reside in the global namespace. We are able to print x from func1 because x is in the global namespace, and we are able to call func1 from func2 because func1 and func2 reside in the same namespace. Remember, anything declared within the global namespace can be used anywhere within the global namespace, including namespaces created within the global scope.

Built-in is a namespace that resides in every Python module that you create because it is literally built in. The built-in namespace resides one step above the global namespace. Even though this module is built into Python, you still have to import it to view its contents, because the word builtins does not reside in the namespace that it includes.

>>> import builtins
>>> [x for x in dir(builtins)]

This example will display the contents of all Python's built in errors, keywords, and functions. Python uses this list when looking up a name that it cannot find in any other namespace. Since this namespace is included in every module's namespace, that means that you can use anything in this list without importing it. The order of namespaces that Python will search for a name in order are local, nonlocal, global, built-in.

The Global Statement

The global statement is what we call a namespace declaration. Global is a reserved word in Python used for special cases where you need to change a global variable from inside a function. Normally, we cannot change the values of global variables from local namespaces. If you declare a variable in a local namespace with the global keyword, the variable will then refer to the global variable, and you are free to change it from the local namespace.

1. x = 10
2. def change_x(value):
3.     global x
4.     x = value
5. print(x)

The global statement within the change_x function makes the variable x inside of the function refer to the x variable declared in the global scope. Now when we pass a value to the function, it will change the value of x in the global scope.

Be careful with global statements! If you are new to programming, you should probably try to avoid using global statements all together, for now. Globals are needed in a few cases, but do not use them unless absolutely necessary. They are notorious for making code difficult to understand and reuse.

local, nonlocal

Local namespace is referring to names that are created within a local scope. Every function has a local namespace; meaning, names declared within the function only reside in in that function's scope. When we declare a variable in a function, it cannot be used outside of that function because it resides in the local namespace.

1. x = 1   # Global namespace variable
2. def local_x():
3.     x = 2   # Local namespace variable
4.     print(x)
5. print(x)

When this function is called, it will print 2 because that is the value of x in the local namespace. After the function, there is a print statement, which also will print the value of x. This will print out 1 because that is the value of x in the global scope, and we cannot access the function's local scope from outside of the function body.

Nonlocal namespace is created whenever we nest a local namespace within another local namespace. A good example of this is called a closure. We won't get into closers here, but basically they are functions nested inside other functions to change the functionallity of one of the functions. Remember, every function has its own local namespace. This means that if we nest a function inside another function, we are nesting a local namespace inside of another local namespace. This creates a nonlocal namespace for the inner function. All names declared inside of the outer function are considered nonlocal variables from the perspective of the inner function.

The Nonlocal Statement

1. x = 10
2. def outer():
3.     x = 3
4.     def inner():
5.         nonlocal x
6.         x = 0
7.     inner()
8.     print(x)
9. outer()
10. print(x)

The first time that we declare x, we set its value equal to 10. This variable lives in the global namespace of this program. We then declare x again within the outer function, this x variable lives within the outer function's namespace and is local to outer. Because the inner is function nested within the outer function, this function, too, can use the x variable declared within outer. Relative to the inner function, the x variable declared in the outer function is a nonlocal variable. Normally, we would not be able to change the value of x from the inner function, but we can because we use the nonlocal keyword in the inner function. The statement "nonlocal x" in the inner function makes the value of x in the inner function refer to x in the outer function. We then change the value of x in the inner function. After the inner function, but still within the outer function, we call the inner function. This will change the value of x in the outer function. Finally, in the outer function, we print the new value of x. The final print statement after the function call to outer will print the global variable x just to show that nonlocal does not impact global variables.
