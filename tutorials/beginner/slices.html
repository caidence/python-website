Slices, indexes

Slices are a convenient way of getting specific data out of an ordered data type. Slices are most commonly used with lists and strings, but can be used with any ordered sequence in Python.

>>> S = 'foo bar'
>>> S[0]
f
>>> S[-2]
a

We can put a number in square brackets after an object name and it will return what is at that index in the object. On the first line, we get the first item in the string S by typing S[0]. S is the object, and [0] is the index of the object we want to return. As you can see, we can use negative indices. -1 is always the last item in the object. In the example, we write S[-2], this returns the second to last item in the S object.

We can also specify what part of the object that we want by using slices which will return a section of the object. Here are a few examples.

>>> S = [0, 1, 2, 3, 4]
>>> S[1:]
[1, 2, 3, 4]
>>> S[:3]
[0, 1, 2]
>>> S[1:3]
[1, 2]
>>> S[:]
[0, 1, 2, 3, 4]

I wanted to create a list to use with this example whose elements are the same as their index to make things a little more clear. These examples demonstrate how to get a section of a list using slices. The syntax is list[(start index):(stop index)]. You just separate the start index from the stop index with a colon. The slice will include the item at the start index, but not the item at the stop index of the slice. If you do not specify a start index, it will default to the beginning of the object. Likewise, if you do not specify an index to stop at, the slice will stop at the end of the object. For the first example of a slice we write S[1:]. This will start the slice at the index 1 and continue until the end of the list because we did not specify an index to stop at. Next, S[:3] will start at the beginning of the list because we did not specify where to start the slice and will end at the second index. It will only go to the second index because Python excludes the end index of the slice. S[1:3] will return all items between indexes 1 and 2 because we start at index 1 and end at index 3 which is excluded from the slice. S[:] essentially just makes a copy of the list because we do not specify where the slice should begin or end.

Python has an optional third index which is called a step. The step is added to the index of each item that is extracted. Like with the first two slice indices, the step can be negative. You add a step by adding another colon after the upper bound slice index. The syntax for adding step to a slice is, list[(start index):(stop index):(step)].

>>> alpha = "abcdefghijklmn"
>>> alpha[::2]
'acegikm'
>>> alpha[1:11:2]
'bdfhj'
>>> alpha[::-1]
'nmlkjihgfedcba'

On line one, we do not specify a lower bound or an upper bound index so the step (third number) will apply throughout the list. Remember, the step is added to each index that is extracted. Because the step is two and the list starts at index 0, alpha[::2] will extract every other item from the string. alpha[1:11:2] practically does the same thing, but because we specify an index to start and stop at, it will only return every other item between the specified indices – in this case 1 and 11. The last example is a little strange, we pass a negative index to the step portion of the slice. [::-1] is a way to return an object in reverse order. In fact, it is one of the fastest ways in python to return an object in reverse order.
