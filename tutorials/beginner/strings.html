The basics

In Python, there are many different data types. One of the most common data types is called a string. Strings are used to store textual information such as your name or text stored in a file. A string is a good example of a sequence which is a positional ordered collection of objects generally read from left to right. Strings are what we call immutable meaning that they cannot be changed in place. String operations create a new string, they never change the origional string. In Python, all objects are either immutable or mutable, but we'll get to this later.

1. # This is okay
2. 'This is a valid string in Python'
3. "This is another valid string"
4.
5. # This won't work
6. 'This is not a valid string"
7. "Neither is this!'

We can create a string using either a pair of single quotation marks or double quotation marks with text in-between. Everything between the pair of quotation marks will be stored in the program as a string. It is important to note that you cannot mix single and double quotes when making a string. The quotation marks must both either be single or both be double but not one of each! The code example demonstrates valid and invalid strings.

Lines one and two in the example are both valid strings because they do not mix and match single and double quotation marks. Nither line six or seven will work because the opening quotation mark is not the same as the closing quotation mark. What about lines one and five? Are these strings too? No, these lines are called comments. Comments are lines that are ignored by the program and are purely used for documentation. Comments are usually used to describe a line of code and can be inline, single-line, or even multi-line (we'll get to multi-line comments in a later tutorial). Single and inline comments are declared using a pound symbol "#". Anything on the line typed after this character will be ignored by the program. Thats enough about comments for this tutorial. Now back to strings!

Escape Sequences

There are certain characters that you cannot use in a string such as \ and quotation marks. If you created your string with single quotation marks you will not be able to put single quotation marks inside of it. Respectively you cannot put double quotation marks inside a string created with double quotes.

1. # This is valid
2. "The boy said 'hello everyone!'"
3. 'The boy said "hello everyone!"'
4.
5. # Now with the escape character
6. "The boy said \"Hello everyone!\""
7. 'The boy said \'Hello everyone\''
8.
9. # Backslash in a string
10. "\\ This is a single backslash"

Escape sequences allow us to put these characters inside of our strings. For instance, if you wanted to put a single backslash in your string, you would have to put two consecutive backslashes in your string "\\". Likewise, if you wish to put single quotes in a single quote string or double quotes in a double quote string, you would have to use the backslash escape character in front of the quotation marks.

Lines two and three work because the quotation marks inside of the string are not the same as the quotation marks used to create the string. On lines six and seven we use the escape character (backslash "\") to get the quotation marks inside of the strings. We have to use the escape character because the quotation marks inside of the string are the same as those used to create the string. If you use the string, the escape character will not show up in it. For example, if you pass one of the strings to Python's built-in print function, the backslash will not appear in the output. Moving on, line ten is how you would put a single backslash inside of a string.

1. # Raw strings in action
2. r"C:\windows\prefetch"
3. R'C:\windows\prefetch'

A neat feature of Python 3 is something called a raw string. Raw strings basically turn the escape character off so that you can use backslashes freely as you please in your strings. This is handy in many cases, for example, using a string as a file path to a location on your computer. We can create a raw string by placing an r right before the opening quote of our string (the r can be upper or lowecase). Lines two and three in this example do exactly the same thing.

Tabs and Newlines

Now would be a good time to quickly introduce tab and newline characters. With Python strings, there is an easy way to code tabs and newlines using the escape character. If you want a newline in a string, you can use "\n" which is the the newline character. Likewise, if you want to code a tab into a string, you would use the tab character which is "\t". The examples demonstrate how to use these sequences in practice.

1. "This string has \t a tab in the middle"
2. "Line 1\nLine 2"

In the example, the first line will have a tab in the middle of it if it is printed or output to the console. The second line will display as two separate lines with "Line 1" on the first line and "Line 2" on the second.

print

Before we go any further, I thought it would be important to introduce you to print(). In Python, print() is one way we can output data to the console. Anything between the braces () is what is output to the screen. For example, if you were to do something like print('Hello'). The program would print 'Hello' to the console. Here are a few examples of printing strings to the console. Side note, any line that starts with >>> in teh examples is a line of code, and any line not prepended with >>> is console output. You will soon figure out that this is a common way to display programming examples.

>>> print("Hello world!")
Hello world
>>> print("Line 1\nLine 2")
Line 1
Line 2

In the code, everything that appears between the () in the print statement is output to the screen. You can see an example from earlier where we used the newline character (\n) to print a newline.

Basic String Operations

>>> print("Hello " + "world!")
Hello world!
>>> print("foo" * 3)
foofoofoo
>>> len('abc')
3

There are a few common string operations that you should be aware of in Python. You can add strings with a "+" (called concatination), multiply a string a finite number of times, and get the length of characters in a string using len() just to name a few. These operations are not specific just to strings. In fact, len is a sequence function meaning that it works on any data type that is considered a sequence in Python. Where as "+" and "*" are overloaded operators of the string class.

In the first example, we use the "+" operator to essentally "add" two strings together. In programming this is called concatination. Next, we see that you can multiply a string by a number, which basically just concatinates the string with itself as many times as the expression states. In the example, we multiplied 'foo' times 3 which output 'foo' three times in a row: 'foofoofoo'. In the last example, we use len, which is short for length, to return the number of characters in the string between the braces (), in this case 'abc'. These are just a few of the most basic operations that you can use on strings. There are many more in the tutorials to come!
