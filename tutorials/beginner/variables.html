The Basics

Variables are incredibly useful in any programming language. You can think of a variable as a placeholder or container for data types. In Python, variables can be changed and used as many times as you want – there are not type restrictions as there are in languages like C and C++. You can create a variable by using the syntax variable_name = data. When you use a variable later in your code, when the code is executed, the variable is replaced with the data you set it equal to previously.

1. # Variable examples
2. list_variable = [1, 2, 3]
3. int_variable = 77
4. float_variable = 1.2

Looking at line three of the example, if later in your code you wanted to use the value 77, you could use int_variable in its place and the program would interpret it as 77. This because we stored that value in the variable int_variable. int_variable is now a container for the value 77, you might think of it now as an alias for the value 77. Variables are very convenient, as you can imagine, especially for larger sets of data such as a large dictionary or a list. In the next section I created some examples of how variables can be used.

Examples

In this section, I will be using the Python's print function to demonstrate how variables work. Hopefully, after reading this section, you will have a solid understanding of how variables work.

>>> name = "Caiden"
>>> print(name)
Caiden
>>> name = "Bob"
>>> print(name)
Bob

On the first line, we create a variable called name and set its value equal to my name, Caiden. You can see that when we use the variable name in our program, that it will replace the variable with the value stored in it. We can change the variable later on if we want, which will override the its current value, which is Caiden. The first time we print the variable, we get the result "Caiden". Next, we change the variable to store the value "Bob", which overrides the old value and replaces it with the new value, Bob. When we print the variable again, it no longer will print "Caiden", it will print "Bob".

Variables can store almost anything, even other variables! In the next example, we can see how this works, as well as a few more tricks when we create variables. We can declare multiple variables on the same line, called multiple variable declaration. You can do this by separating the variables with commas and the data with commas. In multiple variable declaration, the first variable will equal the first object, the second variable will equal the second object, and so on.

>>> a = 1
>>> b = a
>>> print(b)
1
>>> # Multiple variable declaration
>>> a, b = 1, 2
>>> print(a, b)
1 2
>>> # Swapping variable values
>>> a, b = b, a
>>> print(a, b)
2 1

This example starts off with setting a variable a equal to the value 1. We then set a variable b equal to the variable a, and then print the variable b. Because b is a variable that points to another variable, when we print b, it will print the value of the variable that b points to. In this case, b points to a so when we print b, we get back the value of a. Next, we declare two variables on the same line. We make variables a and b, and we set a equal to 1 and b equal to 2. Next, we swap the values of the variables on one line by using multiple variable declaration again. a, b = b, a will swap the values of variables a and b. This works because now a points to the value at b and b points to the value at a. This is a neat feature specific to Python. In other programming languages, swapping variables requires three lines of code.

variable names

Naming variables is pretty straight forward. It is considered best practice to name your variables something meaningful and try to avoid using single letter variable names such as a and b. I use these names often in examples, but you should not use them in practice! There are not too many restrictions on what you can name a variable. The only rules you have to follow are: variable names can only contain uppercase and lowercase letters, underscores, and numbers. Variable names cannot start with a number. Variable names can start with an underscore, but this is used in special cases, which we will demonstrate in a later tutorial. Also, try to avoid naming variables the same as Python classes and functions. For example, if you were to name a variable print, now you would have a variable named print and a function named print, and this can cause conflictions later in your program. One last note, if you name a variable something with multiple words in the name, for instance if you were to name a variable "i love python", the Python convention is to lowercase all letters in the variable name, and to separate the words with underscores. Following convention, your variable would look like this i_love_python. You do not have to follow this convention, but it is strongly recommended!
