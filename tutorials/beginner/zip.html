The basics

Zip is a built-in function in Python. We use it to put the similar index of two or more containers into a single container together. Generally, zip is used with two or more arguments, but it can be used with just one. If you only pass one argument to zip, it creates an object containing single item tuples, each of which contain a single item of the object.

Zip in Use

Like I said earlier, we can pass a single argument to zip or as many arguments as we want. If we pass a single item, zip will create single item tuples of the items in the argument object. If we pass more than one argument to zip, it will create tuples whose length depends on how many arguments we passed to the function. This is because the tuples created by zip contain similar indices of the items.

>>> # Passing a single item
>>> list(zip('Hello'))
[('H',), ('e',), ('l',), ('l',), ('o',)]
>>>
>>> # Passing more that one item
>>> list(zip('Hello', 'world'))
[('H', 'w'), ('e', 'o'), ('l', 'r'), ('l', 'l'), ('o', 'd')]

The first thing to notice is that I wrap the zip call with "list". This is not necessary; I am only doing this in order to view the contents of the zip object that we create from the function. The first time that we call zip, we only pass one argument, a string. Because we do not pass any additional arguments to "zip" our string with, we get back single value tuples containing each index of the string. Remember, single value tuples must have a trailing comma, otherwise they are not a tuple. In the next call to zip, we pass two arguments the strings "hello" and "world". The function will take the first indices of these two strings and pack them in a tuple together. Zip will do this for every pair of indices in the two strings.

Zip is not limited to two arguments, how many objects that we want to zip together is arbitrary. We can zip three, four, or as many objects as we want. Below is an example of passing three objects to the zip function.

chars = ('one', 'two', 'three')
spanish_chars = ('uno', 'dos', 'tres')
ints = (1, 2, 3)
list(zip(chars, spanish_chars, ints))
[('one', 'uno', 1), ('two', 'dos', 2), ('three', 'tres', 3)]

This example in particular I thought would be helpful because the number in each tuple corresponds to its position in the tuple. Zip will package all of the first values into a tuple, all of the second values into a tuple, and all of the third values into a tuple.

Zipping Dictionaries

Zip also makes it easy to create dictionaries on the fly out of lists. If we have a list of keys and values, and wrap the call to zip in a dict container, zip will actually map keys to values for us.

>>> keys, values = ('one', 'two', 'three'), (1, 2, 3)
>>> dict(zip(keys, values))
{'one': 1, 'two': 2, 'three': 3}

On the first line, we create two tuples with the first tuple being all of the values we want in the dict as keys, and the second tuple being the values that we want to match to those keys. There is no special syntax when creating dictionaries with zip, all you need to do is wrap the zip function call with "dict( )".

Let's take it one step further just to make sure that you know how the zip function works. Consider from this point on in this subsection optional reading. Let's create a dictionary from three tuples. We want the vales of this dictionary to be two item tuples. Since, we are given three tuples to begin with, we can use a nested zip call to achieve this. The outer zip will take the dictionary keys as a first argument and a nested call to zip as the second argument. The nested zip will execute first, creating the tuples that we want to pass to the first zip function which will be the values to the keys passed as the first argument to the outer zip call.

>>> keys = (1, 2, 3)
>>> english_chars = ('one', 'two', 'three')
>>> spanish_chars = ('uno', 'dos', 'tres')
>>> dict(zip(keys, zip(english_chars, spanish_chars)))
{1: ('one', 'uno'), 2: ('two', 'dos'), 3: ('three', 'tres')}

We are given three tuples to create a dictionary out of keys, english_chars, and spanish_chars. In Python, nested function calls work from inside to out. This means that the nested zip call will execute before the outer zip call executes. The nested zip creates two values tuples by zipping together english_chars and spanish_chars. These tuples will become the values of the keys passed to the outer zip call. After the outer zip creates two item tuples (the first value being from keys and the second value being a tuple created by the nested zip call), dict will map the first value to the second as key value pairs.

Unzipping

First, we learned how to zip objects together into tuples, now we are going to learn how to use the zip function call to unzip these tuples back into their original state. The items that we "unzip" do not have to have been zipped originally, they just have to follow the syntax of a zipped item.

>>> zipped_list = [('one', 1), ('two', 2), ('three', 3)]
>>> chars, ints = zip(*zipped_list)
>>> chars, ints
(('one', 'three', 'two'), (1, 3, 2))

On the first line, we have a list of tuples. If we unzip this object, we can put all of the first indices into a single tuple and all of the second indices into a separate tuple. Unzipping objects is just the opposite of zipping objects. Notice the asterisk (*) before the object in the call to zip? This is important, we will cover this syntax in a later tutorial, but for now just know that this is how to unzip objects. All of the first indices will be placed in chars and all of the second indices will be placed in ints. When we view the contents of our two newly created tuples (chars and ints), we can see that the first index of each tuple in the original object (zipped_list)is in chars and the second index of each tuple in the original object is in ints.

We are not just limited to unzipping lists of tuples, we can also unzip dictionaries as well. To do this, within the call to zip, we have to call the .items( ) method on the dictionary object. This works because .items( ) will create two value tuples of key value pairs, which, coincidentally, is exactly what we need in order to unzip an object!

>>> zip_dict = {'one': 1, 'three': 3, 'two': 2}
>>> keys, values = zip(*zip_dict.items())
>>> keys, values
(('one', 'three', 'two'), (1, 3, 2))

We start with a dictionary, zipped_dict. We then use multiple assignment to assign keys equal to the tuple containing the dictionary keys after its unzipped and values equal to the tuple of dictionary values after it is unzipped. Like with lists, we must place an asterisk (*) in the call to zip before the dictionary object. Notice that we call .items( ) on the dictionary object in the call to zip. If we do not call .items( ) on the dictionary object, we will get an error because zip will only try to unzip the keys, not the values. When we view the contents of keys and values, we can see that all of the dictionary's keys and values are in two separate tuples.
