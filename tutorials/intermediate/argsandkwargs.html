The Basics

Many people who are new to programming and learned or are learning Python as their first language (myself included), have a rough time understanding *args and **kwargs. I am going to try and clear up some misconceptions in this part of the website. Thought they seem magic, I can assure you that they're not. Before we get in to learning how these two work, there is one thing that you should understand. The names args and kwargs do not matter, it is just convention that we use these names. You could just as easily use *foo and **bar. The only thing about the syntax that is important is the asterisk and the double asterisk, not the names that follow them.

Really just Objects

The title of this subsection is a little misleading but it was much shorter than "Really just variables packaged into objects". That is really all that *args and **kwargs do, is package an arbitrary number of variables into an object. *args will package variables into a tuple and **kwargs will package items into a dictionary. Here is a program to demonstrate this.

>>> # *args, really just a tuple
>>> def example_args(*args):
...     print(args)
>>> example_args(1, 2, 3)
(1, 2, 3)
>>>
>>> # **kwargs, really just a dictionary
>>> def example_kwargs(**kwargs):
>>>     print(kwargs)
>>> example_kwargs(one=1, two=2, three=3)
{'one': 1, 'two': 2, 'three': 3}

With the first function, we use *args in the function definition. The function definition is probably the most common place to use *args and **kwargs. Since we put *args in the function header, every argument that is passed to the function will be packaged into a tuple called args. We can iterate over this tuple in the function. All that example_args does is print the variable args to demonstrate that it is a tuple object.

The second function, example_kwargs, will take all keyword arguments passed to the function and map them into key value pairs in a dictionary. We cannot pass positional arguments to **kwargs just like we cannot pass keyword arguments to *args. We can however, put *args and **kwargs in a function definition which will accept keyword and positional arguments. If we are going to use both *args and **kwargs, *args must be the first argument and **kwargs must be the second. Also, in the function call, positional arguments must appear before keyword arguments.

>>> # *args must be before **kwargs in the function header
>>> def args_and_kwargs(*args, **kwargs):
...     print(args)
...     print(kwargs)
>>> args_and_kwargs(1, 2, 3, four=4, five=5)
(1, 2, 3)
{'four': 4, 'five': 5}

All that args_and_kwargs does is print the tuple and dictionary objects created from the passed positional arguments and keyword arguments. Positional arguments will be packaged into a tuple together and keyword arguments will be a packaged into a dictionary with each argument mapped to its keyword as a key value pair. *args must appear before **kwargs in the function header and positional arguments must be passed to the function before keyword arguments in order for this to work.

Because *args and **kwargs create objects called args and kwargs respectively, we can treat these variables as objects within functions. We can treat args as a tuple within functions because *args packes positional arguments into a tuple. Likewise, we can treat kwargs as dictionary because **kwargs packages keyword arguments into a dictionary with the keyword being the key and its argument being the value.

>>> def max_number(*args):
...     max_num = args[0]
...     for number in args:
...         if number > max_num:
...                 max_num = number
...     return max_num
>>> max_number(1, 2, 3, 4, 15, 6, 7)
15

max_number is a simple function that will return the maximum number from all the positional arguments passed to it. It works by packaging all of the positional arguments into a tuple object called args. We can now use args just like we would use a tuple. This is why we are able to iterate over it in a for loop.

>>> def print_pairs(**kwargs):
...     for key, value in kwargs.items():
...         print(key, 'is', value)
>>>
>>> print_pairs(first_name='John', last_name='Smith')
first_name is John
last_name is Smith

This program is not too exciting; it is just to demonstrate that **kwargs really creates a dictionary called kwargs. All that the function print_pairs does is print the key value pairs of the dictionary created by **kwargs.

Unpacking

When we use *args and **kwargs in the function header, it is called packing variables. This is because we are "packing" values into the variable name that follows the asterisk. We can also use the asterisk syntax when we call a function. This is called unpacking because you are taking an object, i.e. a list, dictionary, or tuple, and "unpacking" its values into function arguments.

1. def print_args(arg1, arg2, arg3):
2.     print('arg1:', arg1)
3.     print('arg2:', arg2)
4.     print('arg3:', arg3)
5.
6. test = (1, 2, 3)
7. print_args(*test)
arg1: 1
arg2: 2
arg3: 3

In this example, we have a function named print_args that will print the values of the three arguments passed to it. After the function, we create a tuple named test. We then unpack the values in test as the arguments to the function in the function call by writing print_args(*test). This will unpack the first index in test as the first argument in the function and so on. Notice how we use the variable name test before the asterisk. This is important and what makes the tuple unpacking work.

Similarly, we can unpack the values of a dictionary as the arguments to a function. If we do this, it is important that the argument names in the function header match the key names in the dictionary that we are unpacking. This will map the dictionary's values associated with that key name to the arguments.

1. def greeting(first_name, last_name):
2.     print(f'Hello, {first_name} {last_name}')
3.
4. person1 = {'first_name': 'Josh', 'last_name': 'Williams'}
5. person2 = {'first_name': 'Logan', 'last_name': 'Smith'}
6.
7. greeting(**person1)
8. greeting(**person2)
Hello, Josh Williams
Hello, Logan Smith

First, we define a function called greeting. This function takes two arguments: first_name and last_name. The function will print a formatted string that greets the person. Following the function, we create two dictionaries, person1 and person2. Notice how the keys in the dictionaries match the argument names in the function header? This is very important and is how we are able to unpack the keys' values as arguments to the function. When we call the function with the dictionary object, we prepend the dictionary name with a double asterisk (**).
