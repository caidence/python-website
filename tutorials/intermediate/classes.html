The basics

If you have read any of the previous tutorials, you've probably seen me use the word "object" to refer to different data types. Really, every object type is a class and datatype just refers to what kind of object something is. We have really been using classes all along! Simply put, a class is a coding structure used to implement our own custom objects. Classes are part of what we call object-oriented programming (OOP). OOP is a more effective way to code because it is reusable and reduces code redundancy. It allows us to define our own objects which we can create our own methods for. Classes are essentially a template to create your own objects that also defines how our program can interact with the objects that we create.

OOP is a form of programming that focuses on using objects to design a program. An object is a component of programming that preforms specific actions (based on the programming language) and interacts with other parts of a program. Classes allow us an easy way to structure our code in a way that makes more logical sense to us as humans. The behavior of objects is determined by methods which are subroutines that have the ability to operate on and manipulate objects and classes.

Why Classes?

Classes are an easy way to structure real objects in the real world around us. There are many advantages to using OOP over functional programming. With using OOP, you can optimize code organization. OOP defines standard ways to easily organize data and procedures in your code. If you use OOP smartly, it will help yourself and others more easily understand what is happening in your code.

Building off of organization, another advantage of OOP is encapsulation. By encapsulation, I am referring to how data is conveniently stored alongside the methods that create and manipulate it. This makes it easy to hide code from the public interface. If we want, we can easily change this hidden code without changing the interface on the public side of the program.

Probably one of the most notable advantages of OOP is inheritance. This is a feature of OOP that allows us to easily build off of code that we have already written. It lets us define data in one place, use it and, or override it in another place if need be. It allows us to extend our own code, or even built-in code. Inheritance makes our programs easily scalable.

OOP makes coding easier on the programmer because it is very flexible and offers great reusability. We are able to "borrow" code from other classes that we have created and use that code in new classes. We are also able to import classes that we write into other programs so that we can use the objects that our classes define in the new code.

Of course, these are not all of the advantages of using OOP. It is a really good idea to start using an OOP approach for your programs. OOP is a pretty ubiquitous concept and is standard in all major programming languages. OOP reduces code maintenance overhead and is probably the most popular paradigm that programmers use today.

Getting Started

Starting off, it is important that you understand the difference between an instance and a class. Classes act as a blueprint for default behavior of instances. You can think of a class as a factory for instances. Instance objects are the objects that your program processes. Every time that you call a class, you will create a new instance of that class. There can exist more than one instance from the same class in a program at the same time and each instance can have its own data. Just remember, classes are templates to create instances and instances are the objects in a program that we interact with.

Here is a class on the most basic level. We define a class by using the class keyword followed by the name of the class. x and y are called class variables. These two variables will be the same for all instances of the class we create unless we explicitly change them change them. If we change either of these variables in the class, it will change the variable in each instance of the class. However, if we change these variables from an instance, it will not change the variables in the other instances.

>>> class Example:
...     x = 10
...     y = 3
>>>
>>> instance_one = Example()
>>> instance_two = Example()

Here, we create a class called Example which has two class variables -- x, and y. x and y will be the same value for each instance that we create of the class. As you can see, we create instances with the format: instance_name = class_name( ). instance_one and instance_two are completely separate objects. If we manipulate one, it will not manipulate the other. Instead of saing that the object type of instance_one and instance_two is string, or tuple, or list; we say that the object type of instance_one and instance_two is Example.

To access the variables of our objects, we use the syntax: instance_name.variable_name. We can change the class variables for all classes by using the syntax: class_name.variable = new_value. By using this syntax, we effectively change the value of the class variable for every instance of the class. Building off of the example from earlier, here is how to access variables from the instance, as well as change the class variables for all instances.

>>> instance_one.x
10
>>> instance_two.x
10
>>> Example.x = 5
>>> instance_one.x
5
>>> instance_two.x
5
>>> instance_two.z = 6
>>> instance_two.z
6
>>> instance_one.y = 45
>>> instance_one.y
45
>>> instance_two.y
3

There is a lot of important information here so let's break it down. On the first and third lines, we access the class variable x by typing the instance name appended with a period followed by the variable name. In the class Example, the class variable x is 10, thus, every instance created from the Example class will have a variable x which equals 10. The same applies for the class variable y. After we see that each instance has a variable x, we write "Example.x = 5". Remember, Example is the name of our class, meaning that this statement will change the value of the variable x in every instance of the class to 5. We see this by viewing the value of the x variable of instance_one and instance_two. Next, we set the variable z of instance_two to equal 6. But wait, Example does not have a class variable named z, so how is this possible? This is completely legal in Python. This line will create a variable named z in instance_two, but not instance_one. Please don't do this, more often than not, there is a better way to write your classes than to create variables this way. Finally, we change the value of y in instance_one. When we now view the value stored at y for both instances, we see that its value in instance_one is now 45, but it did not change the value of it in instance_two.

This is how classes work on the most basic level. There is a lot more ot OOP than assigning variables to objects. We will take a deeper dive into some of the more useful aspects of OOP in later tutorials. This tutorial was just created to briefly introduce you to what we will be working with for many of the tutorials to come.
