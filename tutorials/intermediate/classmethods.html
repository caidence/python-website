The basics

Class methods are slightly different than instances methods. Even though it is possible to call a class method from an instance; generally, you will only see programmers call class methods from the class itself. Class methods belong to the class, not any particular instance. Like with other methods, they do accept a first automatic argument. You can think of this argument much the same as self. Instead of using self, class methods use cls as the first argument. This is because we are passing the class as the first argument to the method rather than an instance.

To make turn a method in to a class method, we can use the @classmethod decorator. Just remember, class methods operate on the class, not an instance. Because of this, two of teh most common use cases for class methods are: changing class variables, and customizing how we are able to initialize instances of the class.

example

Let's make a class called Person. It will be a simple class, but we will use it to demonstrate both; how we can use class methods to change class variables, as well as how we can use class methods to customize how we initialize instances of our class. All people have a first name, last name, and age. People, however, are all from different nations. For the sake of example, we are going assume that our class is being used in the United States and that the instances of people we create live in America. Assuming this, we are going to create a class variable called nation and set its value equal to "America". A class like this would look something like the following.

1. class Person:
2.     nation = 'America'
3.     def __init__(self, first_name, last_name, age):
4.         self.first_name = first_name
5.         self.last_name = last_name
6.         self.age = age
7.
8.     def fullname(self):
9.         return f'{self.first_name} {self.last_name}'
10.
11.     @classmethod
12.     def change_nation(cls, new_nation):
13.         cls.nation = new_nation
14.
15.     @classmethod
16.     def from_string(cls, person_string):
17.         first, last, age = person_string.split(',')
18.         return cls(first.strip(), last.strip(), int(age))

This is a basic class. We can create instances of Person which can all have a different first name, last name, and age. Even though these instance variables can be different; they will all share the nation class variable. We can modify the nation variable with the class method change_nation. To tell Python that a method is a class method, we put the @classmethod decorator directly above the method definition. Like with regular methods, class methods accept a first, automatic argument. Instead of using self, we use cls. This is because class methods work on the class, not a particular instance. This first argument will refer to whatever the class name is, in this case, Person.

The first class method will change the class variable nation. Notice how we use cls.nation in the method to access this variable? This is the same as saying Person.nation because the class name is Person. The next method from_string customizes how we can create instances of the person class. For example, say we have a string like this: 'Caiden, Thompson, 19'. We can create an instance of the class from this string by using the from_string method. How this method works is: it takes a string, creates first, last, and age variables by parsing the string and passes them to the person class by using the cls argument, and returns the instance created. Let's take a look at an interactive session.

>>> p1 = Person('Will', 'Schield', 22)
>>> p1.fullname()
'Will Schield'
>>> p2 = Person.from_string('Caiden, Thompson, 19')
>>> p2.age
19
>>> p2.fullname()
'Caiden Thompson'
>>> Person.change_nation('Canada')
>>> p1.nation
'Canada'
>>> p2.nation
'Canada'

Here, we first create an instance of Person called p1. Next, we call the fullname method on this instance just to demonstrate what the method does. After that, we create an instance of Person by using the from_string class method. This method works by taking a string that we pass to it; parsing the string for the first name, last name, and age; and passing the variables from the parsed string to the class and returning the instance created. When we create an instance of Person from the class method, we have to explicitly call it from the class like: Person.from_string(). This is because class methods do not operate on instances, they operate on the class.

Like with the from_string class method, you should call the change_nation class method directly from the class rather than an instance. As you can see, when we call Person.change_nation, it will change the nation for all instances of the class. This will change the nation class variable for all current and future instances that we create of Person.
