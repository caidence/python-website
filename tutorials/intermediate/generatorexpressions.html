The Syntax

Generator expressions are somewhere between a generator function and a list comprehension. A generator has the functionality of a generator object with the syntax of a list comprehension. The only difference in syntax between generator expressions and list comprehensions is with generator expressions, you wrap with ( ) rather than [ ]. Generator expressions support for loop nesting and if statements. Like generator functions, generator expressions create a generator object that we can iterate over in our program. One call thing about generator expressions is, because they are wrapped in ( ), we do not need to wrap them in parentheses in function calls.

The functionality of generator expressions is exactly the same as generator functions. Because I covered what generators actually do in the previous tutorial, I will not discuss much how generators work on this page. This tutorial assumes you have read the previous one and is aimed at generator expression syntax rather than what the generator is actually doing behind the scenes.

The Basics

As mentioned earlier, generator expressions create generator objects just like generator functions. The difference is, generator expressions can be coded on one line like list comprehensions. Let's take a look at a few examples where I demonstrate a generator function and create a generator expression that is equivalent to the function.

1. test_list = [1, 2, 3, 4]
2.
3. # Generator function
4. def gen_squares(array):
5.     for num in array:
6.         yield num ** 2
7. example = gen_squares(test_list)
8.
9. # Generator expression
10 example = (num ** 2 for num in test_list)

Lines 7 and 10 achieve the exact same result. The difference is, line 7 creates a generator object by calling a generator function where as line 10 is a generator expression. As you can see, generator expressions have the same syntax as list comprehensions with the difference being that the expression is wrapped in ( ) rather than [ ]. Because generator expressions create generator objects just like generator functions, we can iterate over generator expressions too. We can even replace where we would normally place a variable in our code with a generator expression if we want. A way of doing this is by replacing variable names in for loops with a generator expression.

>>> for x in (x ** 2 for x in range(4)):
...     print(x, end=' - ')
0 - 1 - 4 - 9 -

More often than not, we put a variable referencing an object in the for loop header: for x in variable:. In this example, we see that we can replace where we would usually put a variable with a generator expression. This works because a generator expression creates a generator object. Remember, generator objects are iterators so when we put a generator object directly in the header of a for loop, the loop will automatically iterate over the generator object created by the expression.

Nested Generators

Believe it or not, we have been working with generators in many of the previous tutorials. Python's range built in function actually returns a generator object. This is why we can get away with putting massive numbers in a function call to range. Any time that we use range in a generator we are actually nesting a generator within a generator. Nesting generators has the advantage of saving memory. This is because they will execute without creating multiple lists that would need to be stored in memory.

>>> # Nested list comprehension
>>> [x ** 2 for x in [x ** 3 for x in (1, 2, 3, 4)]]
[1, 64, 729, 4096]
>>>
>>> # Nested generator expression
>>> list(x ** 2 for x in (x ** 3 for x in (1, 2, 3, 4)))
[1, 64, 729, 4096]

Both the nested list comprehension and the nested generator expression have the same effect. Both also have advantages and disadvantages. Let's start with the list comprehension. The major advantage of the list comprehension is its speed. List comprehensions are faster than generator expressions because the list is loaded into memory before the computation begins on the object. The main disadvantage of list comprehensions is; they take up a lot more memory. This is because the entire list object is loaded into memory before the program can execute tasks on the list. This is fine for small lists, but becomes a major disadvantage for large list objects. Generators pretty much have the inverse advantages and disadvantages of list comprehensions. An advantage of generator expressions is; they consume much less memory. This is because as we iterate through a generator object, each element of the object is only loaded into memory as needed. The entire object is not loaded all at once. A disadvantage of generator functions, as you can probably guess by this point, are slower to iterate over than the list object created by comprehensions. This is because the expression actually has to execute with each iteration of the generator object.
