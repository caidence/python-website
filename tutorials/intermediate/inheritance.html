The basics

Inheritance is probably one of the most notable features of OOP. It allows us to build new classes that take code from classes that we have already made. Inheritance makes programs more flexible, versatile, and expandable. With inheritance, we can create base classes, which will act as a common functionality for all future classes. If we program this way, we will not have to write the same method in more than on class because the method can just be inherited if we want!

Generally, when a class inherits from another, it will acquire all properties and methods of the parent class. This is what I mean when I say that OOP is a way to create classes from existing classes. We have really been using inheritance all along believe it or not. Every class that we create actually inherits from a class called object. This is the base class that all objects in Python inherit from. Writing: class Test: is the same as writing class Test(object):. This is the syntax used for inheritance in Python, as you'll see very soon!

Getting Started

When one class inherits from another, it will acquire most of the methods from the class that it inherits from. A few of the methods that are not inherited are constructors, destructors, and Python's __init__. To mitigate a child class not calling its parent's __init__ method, we can call the parent's __init__ method explicitly from the child's __init__ method. Let's look at an example to show how this works, as well as how a child inherits methods from its parent class.

1. class Parent:
2.     def __init__(self, x=0, y=0):
3.         self.x = x
4.         self.y = y
5.
6.     def do_stuff(self):
7.         total = 0
8.         for v in self.__dict__.values():
9.             total += v
10.         return total
11.
12.
13. class Child(Parent):
14.     def __init__(self, x=0, y=0, z=0):
15.         Parent.__init__(self, x, y)
16.         self.z = z

The Parent class accepts two arguments: x and y. Parent also has a method: do_stuff that will add all of the instance variables together and return the total. There is also a class called Child which inherits from Parent. To make Child inherit from Parent, we have to put Parent in the parentheses following the Child class definition like this: class Child(Parent):. Because Child inherits from Parent, we will also be able to call the do_stuff method from an instance of Child. A key point to notice is the __init__ method in Child. Because Child accepts some of the same arguments as Parent, we want to use the __init__ method in Parent for those arguments. We do_stuff this by explicitly calling the Parent's __init__ in the Child's __init__ method and passing the similar vales to the explicitly called parent __init__. For instance, in the example, both Chile and Parent accept the arguments x and t. Because these arguments are the same in both classes, we want to initialize these two arguments by using the __init__ method in Parent. To do this, we can write: Parent.__init__(self, x, y). This will initialize the variables with the Parent's __init__, but we can still treat them as a part of the Child instance, because they are. We can still access these variables from child instances by writing: child)instance.x, or; child_instance.y.

If we create an instance of Child, we can do everything that is defined in the Parent class. The difference between Parent and Child is that, Child has a third instance variable – z. You can think of Child as an extension of Parent. Child will act the same as parent, but it has another instance variable.

>>> test = Child(x=1, y=2, z=3)
>>> test.x
1
>>> test.y
2
>>> test.z
3
>>> test.do_stuff()
6

First, we create an instance of Child called test. As you can see, we can access the x, y, and z instance variables of instances of Child just like we can with instances of Parent (excluding z). Since Parent does not have an instance variable called z, we have to explicitly set it in the initialization method of Child. However, if we can, we want to initialize Child classes with their Parent's initialization functions. Even though we did not explicitly create a do_stuff function in the Child class, we can still call it on child instances because it is inherited from the Parent class.

Super()

There is, however, an alternative to explicitly calling methods from the parent class by their name. We can use the super() function to do this for us. We use the syntax: super().method_name(), to call the parent's method that is named method_name. The advantage of using super over explicitly writing the name of the method in the Parent function is: it eliminates the need to declare certain characteristics of a class. If a class inherits from more than one other class and, for instance, needs to initialize variables that are inherited from each parent class, super will call the __inti__ method from each parent class in one statement rather than having to call each parents' __init__ explicitly.

When you can, you will want to use super in place of directly calling methods from the Parent class. It will reduce maintenance overhead, improve readability, and reduce code redundancy. For instance, in the last example, we had the opportunity to use super. In the __init__ method of Child, because Child and Parent have two of the same instance variables -- x and y. We could call super instead of explicitly calling __init__ from Parent. Here is a look at what the before and after Child class would look like.

1. # Without using super()
2. class Child(Parent):
3.     def __init__(self, x=0, y=0, z=0):
4.         Parent.__init__(self, x, y)
5.         self.z = z
6.
7. # With using super()
8. class Child(Parent):
9.     def __init__(self, x=0, y=0, z=0):
10.         super().__init__(x, y)
11.         self.z = z

These two definitions of Child will work exactly the same. The difference is -- one calls the Parent __init__ method explicitly by using the name of the Parent class (Parent.__init__(self, x, y)), and one calls the Parent __init__ method by using super() (super().__init__(x, y)). Notice how when we use super to call the parent's initialization method, that we do not have to pass self as the first argument to __init__. We do, however, have to pass self when we write: Parent.__initn__(self, x, y). Python's super function is smart enough to where we can exclude the self argument and it will still work the same as calling the parent __init__ by name with the self argument.

We're all people; right?

I wanted to take the opportunity to look at a slightly more sophisticated and larger example of a program that uses inheritance. A simple way to demonstrate this is by using people. We all have different relationships with different people. For instance, we all have friends, family, co-workers, classmates, etc. Even though our relationship is different between each person, they're all still people. This makes for an excellent example to use inheritance. All people, despite their relation to us, have many of the same attributes such as a first name, last name, and age. We can use these common attributes to create a base class, Person. The relationship that we have with each person will all have these attributes in our base class. We can then create a subclass that inherits from Person for people who have different types relationships with us. For instance, Employee, Friend, and Family. These classes can each have their own unique attributes and methods, but still have the same attributes and methods as our base Person class.

1. class Person:
2.     def __init__(self, first_name, last_name, **kwargs):
3.         self.first_name = first_name
4.         self.last_name = last_name
5.         for key, value in kwargs.items():
6.             setattr(self, key, value)
7.
8.     def full_name(self):
9.         return f'{self.first_name} {self.last_name}'
10.
11.
12. class Coworker(Person):
13.     def __init__(self, first_name, last_name, salary=50000, **kwargs):
14.         super().__init__(first_name, last_name, **kwargs)
15.         self.email = first_name[0].lower() + last_name.lower() + "@example.com"
16.         self.salary = salary
17.
18.     def give_raise(self, percent=1.02):
19.         self.salary *= percent
20.
21.
22. class Friend(Person):
23.     def __init__(self, first_name, last_name, phone=None, **kwargs):
24.         super().__init__(first_name, last_name, **kwargs)
25.         self.phone = phone
26.
27.     def change_phone(self, new_number):
28.         self.phone = new_number
29.
30.
31. class Family(Friend):
32.     def __init__(self, first_name, last_name, phone=None, address=None, **kwargs):
33.         super().__init__(first_name, last_name, phone, **kwargs)
34.         self.address = address
35.
36.     def change_address(self, new_address):
37.         self.address = new_address

There is a ton of code here, so let's take it line-by-line. On the first line, we create a class called Person. This is going to be our base class that all other classes will inherit from. This class defines instance variables called first_name and last_name. All other classes will have these variables as well and will set them by calling the __init__ method from Person. In Person, there is also a method called full_name. This will return the full name of the person and will work on instances created from any class.

The next class is called Coworker and inherits from Parent. Because of this, we are able to call the full_name method from instances that we create of Coworker. Notice the initialization method in Coworker. Because, first_name and last_name instance variables are required for both Coworker and Person, we want to initialize those variables with the Person __init__ method. We do this by directly passing the variables from the init method in Coworker to the super().__init__() method. Coworker extends Person. The difference is, Coworker accepts an additional argument – a salary. There is also an additional method in Coworker. give_raise will raise the salary by 2%.

The third class is called Friend. Friends are also people (believe it or not) and that iw why the Friend class inherits from Person. Because Friend inherits from Person, we are able to call the full_name method in the Person class definition from instances of Friend. Like Coworker, Friend has all of the same attributes as Person, it just extends the abilities of Person. Also, like Coworker, Friend creates instances variables called first_name and last_name. Again, we want to create these instance variables from the parent's __init__ method.

The final class in this program is called Family. This class varies slightly from Coworker and Friend in its inheritance scheme. We want this class to inherit from both Person and Friend. We need it to inherit from Person so that we can easily define first_name and last_name instance variables and still get the functionality of Person's full_name method. We also want to use the change_phone method defined within the Friend class. To get the functionality of both, we can choose to inherit from the Friend class. This works because Friend inherits from Person, giving us the functionality of both classes if we inherit from Friend. In Family's __init__ method, when we call super().__init__, phone will be initialized from Friend and the Friend class will pass first_name and last_name to Person so that they can be initialized there.
