The basics

In Python, there are several special method names that are reserved for special use cases. __init__ is one of them (pronounced dunder init). We use __init__ to initialize objects when we create them in Python. Most OOP languages have something called a constructor, which is a special method used to initialize instances created from a class. Python's constructor technically is not __init__, but many people use it this way. It is called automatically when we instantiate (create) and object. We can pass values to it to prepare instance objects for use. __init__ can accept arguments which, often times, are used to set instance variables. In previous tutorials, we learned about class variables which are the same for all instances. Instance variables have the same name across all instances, but they can have different values depending on what we pass to the init method.

dunder init

Usually to make things simple, people list __init__ as the first method of the class. This is not necessary, but it greatly improves code readability. Like with many other methods, __init__ does accept the instance as a first argument by using self as the first argument. This is because we are initializing instances, not the class itself. We can pass arguments to __init__ just like we can to other methods. Often times, you'll see the scheme: self.argument = argument used in __init__. This is how we set the arguments that we pass to it to be instance variables. Remember, self is referring to the current instance. This makes "self.argument = argument" refer to the argument of the current instance and not of other instances. Let's create a class with __init__ to demonstrate how this works.

1. class Person:
2.     def __init__(self, first_name='', last_name=''):
3.         self.first_name = first_name
4.         self.last_name = last_name
5.
6.     def full_name(self):
7.         print(f"{self.first_name} {self.last_name}")
8.
9.     def email(self, email_domain="example.com"):
10.         email = self.first_name[0].lower() + self.last_name.lower()
11.         domain = "@" + email_domain
12.         print(email + domain)

On line two, we have defined a method called __init__. Remember, this method is called automatically whenever we create an instance of the class. We pass arguments to __init__ when we create an instance in the parentheses of the class name. i.e. "instance = class_name(arg1, arg2)". arg1 and arg2 are passed to the init method by position in this case. The first position of init is reserved by self, so arg1 will technically be passed to the second position. Back to the example, notice lines three and four. This is setting the arguments of the instances that we create by creating instance variables. self refers to the instance being created and is appended with a period followed by the variable name that we want. Instance variables can be accessed by any method that uses self as its first argument. This is how we are able to access the variables from the methods full_name and email. Also notice that we can pass default values to the arguments in init. This is useful for making our classes more robust.

Let's take a look at an interactive session with the Person class we created in the last example. We will create two instances of Person to get a grasp on how init works.

>>> p1 = Person("Bill", "Nye")
>>> p2 = Person("John", "Doe")
>>> p1.first_name()
Bill Nye
>>> p2.full_name()
John Doe
>>> p1.email("gmail.com")
bnye@gmail.com
>>> p2.email()
jdoe@example.com

First we create two instances of Person -- p1 and p2. As you can see, we pass arguments to __init__ through the creation of objects. This is done by putting the arguments in the parentheses after the object name. We can then call the methods on the objects instances that we have created and the output will change based on the instance variables that we created with __init__.

First, we call the full_name function on p1 which just prints a formatted string of self.first_name and self.last_name. I've said it many times and I'll probably say it many more because I don't want it to be as hard to learn for you as it was for me. self refers to the instance that we are calling the method on. We have to use the syntax "self.variable_name" to get the variable of the particular instance that we are working with. Back to the example, next, we call the full_name function on p2; which will print a formatted string of the self.first_name and self.last_name variables that we created in __init__. Finally, we can create email addresses for the instances that we create with the email method. This method accepts an optional argument, an email domain (icloud.com, gmail.com, yahoo.com, etc.). If we do not pass a domain to the method, it will default to "example.com". This method works by manipulating the self.first_name and self.last_name variables that we created with __init__ into a valid email address and appending an "@domain" to the end of it. When we view the output of this method, again, we can see that it depends on the instance variables that we created in __init__.

dunder dict

A useful special method is __dict__. We can call __dict__ on any instance, it will return a dictionary of key value pairs containing a mapping of instance variable names and their values. Let's look at an example to further demonstrate this.

>>> class Test:
...     def __init__(self, arg1, arg2):
...         self.arg1 = arg1
...         self.arg2 = arg2
>>>
>>> t1 = Test(1, 2)
>>> t2 = Test(3, 4)
>>> t1.__dict__
{'arg1': 1, 'arg2': 2}
>>> t2.__dict__
{'arg1': 3, 'arg2': 4}

We start by creating a very simple class called Test. We then create two instances of Test called t1 and t2. I passed different values to these two instances to demonstrate that __dict__ returns a dictionary of instance variables that you call it on, meaning, it will return different values for instances if the variables are different. We do not put parentheses after __dict__ because it is not callable, it just returns a dictionary object of variables and the values they store. If we did have class variables in Test, they would not appear in the dictionary returned by calling __dict__. If we call __dict__ on an instance, the dictionary returned will only contain instance variables. Calling __dict__ on instances looks abnormal: instance.__dict__, but it is a pretty handy feature of Python.

The reason that I introduced __dict__ in this article, is so that I can easily demonstrate how setattr works in the next subsection. Just remember, __dict__ returns a dictionary of instance variables. Because this method returns a dictionary, we can treat the call to it like a dictionary object.

setattr

We can easily create an arbitrary number of instance variables by using **kwargs along with setattr in the __init__ method of a class. When used in this context, setattr will take the values stored in a dictionary and map them to instance variables. You can think of setattr kind of like the opposite of __dict__. Instead of making a dictionary of instance variables, we can use it to create instance variables from a dictionary. **kwargs is an easy way to package an arbitrary number of keyword arguments into a dictionary. This makes it easy to set as many instance variables that we want!

1. class Test:
2.     def __init__(self, arg1, arg2, **kwawrgs):
3.         self.arg1 = arg1
4.         self.arg2 = arg2
5.         for key, value in kwargs.items():
6.             setattr(self, key, value)
7.
8.     def view_instance_variables(self):
9.         for key, value in self.__dict__.items():
10.              print(key + ":", value)

Don't worry too much about the method view_instance_variables. All it does is: get the instance variable dictionary of itself and print all the variable names along with their values. What we want to focus on here is lines five and six. The for loop will iterate over every key value pair created by the **kwargs dictionary. Line six is what I wanted to show you. This will create instance variables from the kwargs dictionary. setattr accepts three arguments: (object, attribute, value). We are passing self as the object so that it will initialize the instance with variables that we pass as keyword arguments. Attribute, you can think of as the name of the variable that we want to set. And value, obviously, is the value of the variable that we want to set.

Let's have a look at an interactive session to get a feel for how this works. Remember, just because **kwargs is an argument does not mean that we have to pass any values to it.

>>> t = Test(1, 2, arg3=3, arg4=4)
>>> t.view_instance_variables()
arg1: 1
arg2: 2
arg3: 3
arg4: 4

It is important that keyword arguments are passed after positional arguments when creating instances. Notice that __init__ only has two required arguments -- arg1 and arg2. We, however, can pass an arbitrary number of keyword arguments because of lines five and six combined with **kwargs in the method's definition. When we call view_instance_methods, we see that, lines five and six of the class turned the kwargs dictionary in to instance variables.
