The Basics

In Python, operators either work, or don't for built-in classes (objects). However, they behave differently depending upon the class that you use them on. For example, + is a Python operator. It will add two ints together, concatenate two strings, and merge two lists. This is an example of operator overloading. We can create methods that will allow us to use these operators on instances of our own class by using several special methods.

Each operator is actually a method. That is why they behave differently in different contexts. Defining methods for operators (+, -, *, etc.) is known as operator overloading. For example, to use + with custom objects that we create from user defined classes, the class must contain a method called __add__.

There are a ton of magic methods that Python provides for operator overloading. Operator overloading is not just limited to mathematical operators. Below is a list of some of the most commonly overloaded methods.

# Common operator overloading methods
+          __add__(self, other)
*          __mul__(self, other)
-          __sub__(self, other)
%          __mod__(self, other)
/          __truediv__(self, other)
**         __pow__(self, other)
<          __lt__(self, other)
<=         __le__(self, other)
==         __eq__(self, other)
!=         __ne__(self, other)
>          __gt__(self, other)
>=         __ge__(self, other)
[index]    __getitem__(self, index)
in         __contains__(self, value)
len        __len__(self)

This is not a comprehensive list of every operator overloading function. This is just a list of some of the most common operator overloading methods. There are many more operator overloading methods than just the ones listed here. Each operator overloading methods is a Python magic method, and thus will start and end with double underscores. In this list, the operator that will call the method is on the left, and the method that is called when the operator is used is listed next to it on the right. To get a good feel for how these magic methods work, let's take a look at a small example. We will create a Circle class that defines a few of the more common operator overloading methods listed here.

Example

Let's define a circle class that accepts the radius as an argument. This class will be able to calculate the diameter, area, and circumference based on the radius. Let's also write this class so that we can compare it to other instances of the same class. We can use operator overloading to achieve the comparison part!

1. class Circle:
2.     def __init__(self, radius):
3.         self.radius = radius
4.
5.     def diameter(self):
6.         return self.radius * 2
7.
8.     def circumference(self):
9.         return 2 * 3.1415926 * self.radius
10.
11.     def area(self):
12.         return 3.1415926 * self.radius ** 2
13.
14.     def __add__(self, other_object):
15.         return Circle(self.radius + other_object.radius)
16.
17.     def __lt__(self, other_object):
18.         return self.radius < other_object.radius
19.
20.     def __gt__(self, other_object):
21.         return self.radius > other_object.radius
22.
23.     def __le__(self, other_object):
24.         return self.radius <= other_object.radius
25.
26.     def __ge__(self, other_object):
27.         return self.radius >= other_object.radius
28.
29.     def __eq__(self, other_object):
30.         return self.radius < other_object.radius
31.
32.     def __ne__(self, other_object):
33.         return self.radius > other_object.radius

Each method in this class that starts and ends with a double underscore (excluding __init__) is an operator overloading method. This means that we are now able to use operators on instances that we create of the Circle class and in doing so, the corresponding operator method will be called. For instance, if we create two instances of Circle called c1 and c2 and use the plus operator on them (+) like this: c1 + c2, this will call the __add__ method of the Circle class passing c1 as self and c2 as other_object. If we have the two same instances of Circle and use the equality (==) operator on them: c1 == c2, then the __eq__ method will be called on c1 passing itself as the first argument along with c2 as other_object.

Let's look at an interactive session to demonstrate some of the properties of the Circle class.

>>> c1 = Circle(5)
>>> c2 = Circle(3)
>>> c1 > c2
True
>>> c1 != c2
True
>>> c1 == c2
False
>>> c3 = c1 + c2
>>> c3.radius
8
>>> c3.area()
201.0619264

First, we create two separate instances of circle: c1, and c2. On the third line of this session, we can see that we are able to use the greater than operator to compare our two instances. This was made possible when we defined the __gt__ method within our Circle class. When we use the greater than operator on instances of Circle, it will call the __gt__ method defined in the class. Next, we use the not equal operator to compare the two classes which is defined in Circle by the method __ne__. Likewise, we are able to add two instances of Circle together because we defined the __add__ method in teh Circle class.

When we "add" two instances of circle, it will add together the radius instance variables of both classes, pass it to the Circle class, and return the newly created instance of Circle. This is why setting the variable c3 equal to: c1 + c2, will set c3 equal to the newly created instance of Circle that is returned by __add__. On the next few lines, we can see that c3 is indeed a completely separate of Circle. We are able to call all Circle's methods on c3 because of this.
