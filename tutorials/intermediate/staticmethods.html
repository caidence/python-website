The Basics

Sometimes, there are methods that you want to create that are closely related to a class, but do not need the class itself or an instance of the class to operate. Static methods do not know anything about the class necessarily, they just work with, or alter parameters of the class. Static methods do not require any arguments, but you can pass arguments if you'd like. They do not need the class to work on or an instance to work on. Because of this, they do not need a self or cls argument. We create static methods by using an @staticmethod decorator directly above the method definition. If you have a function or method that is closely related to a class, but does not need to alter or use the class or any instances of the class, then it would probably be a good idea to make is a static method.

example

There are few use cases for using static methods so it was hard of me to think of an example to show here, but Corey Schafer had a great example on his YouTube channel, so I am going to do something similar. Say we have a class to keep track of our workers. All of our workers want us to add functionality to the class so that it will tell them whether it is a work day or not. Work days are Monday through Friday. We do not need any variables from the class or any instance to tell our workers whether it is a work day or not. This is, however, closely related to our worker class, so we can create a static method that tells employees if the current day is a work day or not. This class could look something like the following.

1. class Worker:
2.     def __init__(self, first, last, badge=None):
3.         self.first = first
4.         self.last = last
5.         self.badge = badge
6.
7.     @staticmethod
8.     def is_work_day():
9.          from datetime import datetime
10.         if datetime.today().weekday() < 5:
11.             return True
12.         return False

The focus of this example will be the static method is_work_day. This method will return True if the current day is a work day and False otherwise. Line nine imports a module called datetime that has a method that will check the current day for us. on line ten: datetime.today().weekday() will return what date it is in the form of a number. Monday is 0, Tuesday is 1 ... Sunday is 6. We check to see the value of this number in the if block. If the number is between zero and five, that means it is a weekday, thus, making it a workday.

This method does not require any information from the class or any of its instances. All it needs to know in order to operate is what the current day is. What day it is unrelated to the worker class, but workdays are related to the class. This method can be called on any instance that we create of the class, as well as the class itself.

>>> w1 = Worker('Nick', 'Doe')
>>> w1.is_work_day()
True
>>> w2 = Worker('John', 'Smith')
>>> w2.is_work_day()
True
>>> Worker.is_work_day()
True

In this interactive session, we create two instances of Worker. This is to demonstrate that static methods work the same across all instances of a class. As you can see, when we call the static method from each instance, it will give us the same result (I ran this code on Monday, that's why it returns True for me). Going a little further down, we can see that we can also call the static method directly from the class itself. This makes sense because static methods do not require an instance or a class to operate, that's why they can be called, and work the same when called from instances or the class.
