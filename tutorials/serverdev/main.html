server-dev docs

Python 3 | Tutorials | Algorithms

short description

This module is a Python wrapper for vCenter Management RESTful API 6.5. The goal of this module is to act as a Python bridge for managing vCenter, esx servers, and virtual machines. This module works on Linux and Windows, and supports Python 2 and 3.

important

This module currently only supports esx servers that are running ESXi 6.5. Aside from the ESXi version, some functions and methods require ssh access and esx shell to be enabled on ESXi serveres in order to function properly.

installation

# Windows install
python -m pip install server-dev

# Linux install
pip install server-dev

# clone repo
git clone https://github.com/Caidentp/server-dev

I published server-dev to PyPI, so you can install it as you can any other PyPI package – with pip. I have also included a requirements.txt file for convinence, but if you do not know the install directory, look to the example on how to install server-dev requirements. The only depandancies for server-dev are paramiko, json, and requests. If you do not want to install the module with PyPI, you can clone the repository from here.

dependancies

# PyPI package names to install
json
requests
paramiko

server-dev has some functionallity that relies on ssh and esxcli commands. I used paramiko as my ssh module to make server-dev cross-platform. To call the RESTful API, I used the requests and json Python modules.

quick start guide

# authenticate to vCenter server
from serverdev import *
VCenterSession('your vCenter ip', 'your username', 'your password')

Before we can interact with any of the objects in vCenter, we have to authenticate to the vCenter server. This is done by instantiating the VCenterSession class with your vCenter credentials. Almost all functions and methods in this module require you to be authenticated to your vCenter server.

Now we can interact with esx servers and virtual machines by instantiating them. With the class defined in server-dev.

# instantiate esx
host = EsxServer('esx ip address', 'username', 'password')
virtual_machines = host.get_vms()

Now that we have our vCenter objects, we can call methods on them to control the physical objects.

host.enter_maint_mode()
for vm in virtual_machines:
    vm.poweroff()

The example puts the esx server into maintainence mode and will power off all of its virtual machines.
